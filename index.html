2. D3 is a useful library for making the data flow from data parsing to inputting it into a graph type a very simple process. It has a lot of built-in types of graphs that don't have to be coded from scratch. In terms of practicality and utility, d3 is very well documented and commonly used, so examples are readily available and easy to comprehend and learn from.

3. Because data is so complex, it can be limiting to try to fit data into the necessary form that is required for the default d3 built-in graphs. In some contexts, it is better to just build a graph from scratch. d3 is also complex and very robust in terms of how many options there are, so in that sense, it can be overwhelming for both the person writing the code and for the user, who can be overwhelmed with over-specified graph features. Visualization without standard graphs, and using something more expressive can be a reason to not use d3.
